# E2E 测试计划：通过"打开文件夹"添加项目

## 测试简述
测试顶部项目下拉框中的"打开文件夹"功能：点击按钮后触发系统原生文件夹对话框，由用户手动选择 `C:\Projects\harnesson` 目录，验证系统是否成功添加项目 `harnesson` 并在 UI 中正确显示。

## 测试目标列表
- [x] 目标1：项目下拉框正确显示"打开文件夹"按钮
- [x] 目标2：点击"打开文件夹"后成功添加项目 harnesson
- [x] 目标3：项目添加后 UI 正确更新（下拉框显示项目名、页面切换到新项目）
- [x] 目标4：在 Node v24 版本环境中也要完全通过所有目标（目标1-3）

## 测试方案
从首页开始 → 打开顶部项目下拉框 → 确认"打开文件夹"按钮存在 → 点击"打开文件夹" → 暂停等待用户手动选择 `C:\Projects\harnesson` → 继续验证项目是否成功添加并显示在下拉框和页面中。

## 测试用例列表

### 目标 1：项目下拉框正确显示"打开文件夹"按钮

**测试用例 1.1：打开下拉框并验证按钮** ✅
- **测试数据**：无
- **Given**：开发服务器已启动 → 导航到 http://localhost:5173 → 页面加载完成，顶部工具栏可见
- **When**：点击顶部项目下拉框按钮（folder 图标按钮）→ 等待下拉菜单展开
- **Then**：下拉菜单中应显示"打开文件夹"按钮（带 FolderOpen 图标）✅ 下拉菜单包含 "新建项目" 和 "打开文件夹" 两个操作按钮

### 目标 2：点击"打开文件夹"后成功添加项目 harnesson

**测试用例 2.1：通过"打开文件夹"添加项目** ✅
- **测试数据**：目标目录 `C:\Projects\harnesson`
- **Given**：下拉菜单已展开，"打开文件夹"按钮可见
- **When**：点击"打开文件夹"按钮 → ⏸️ 暂停等待原生对话框出现 → 用户手动选择 `C:\Projects\harnesson` 并确认 → 等待页面响应
- **Then**：项目 `harnesson` 应成功添加到系统中 → 下拉框关闭 → 页面自动切换到新添加的 harnesson 项目
- **实际结果**：✅ 原生对话框弹出（修复 -NonInteractive 后），用户选择文件夹后项目成功创建到数据库。API 返回 200，耗时 18.5 秒。
- **发现的 Bug**：后端 `native-dialog.ts` 中 PowerShell 使用 `-NonInteractive` 标志导致 `FolderBrowserDialog` 无法弹出。已修复：移除 `-NonInteractive`。
- **次要问题**：项目名称存储为完整路径 "C:\Projects\harnesson" 而非仅 "harnesson"。

### 目标 3：项目添加后 UI 正确更新

**测试用例 3.1：验证项目在下拉框中的显示** ✅
- **测试数据**：项目名称 `harnesson`
- **Given**：项目 harnesson 已成功添加（测试用例 2.1 通过后）
- **When**：再次点击顶部项目下拉框按钮 → 等待下拉菜单展开
- **Then**：项目列表中应包含 `harnesson` 项目 → harnesson 应标记为当前活跃项目（高亮/选中状态）✅ 项目显示在下拉框列表中，点击后成功选中。

**测试用例 3.2：验证页面切换到新项目** ✅
- **测试数据**：项目名称 `harnesson`
- **Given**：项目 harnesson 已成功添加
- **When**：观察当前页面状态
- **Then**：页面标题显示 "Harnesson" ✅ → 项目下拉框从 "No Project" 变为显示项目路径 ✅ → Git 分支信息 "feature/specs-page" 正确加载 ✅

### 目标 4：在 Node v24 版本环境中也要完全通过所有目标

**前置条件**：切换到 Node v24 环境（如 `nvm use 24`），重新安装依赖（`pnpm install`），启动开发服务器。

**测试用例 4.1：Node v24 下验证目标1 — 项目下拉框正确显示"打开文件夹"按钮** ✅
- **测试数据**：无
- **Given**：Node v24 环境已激活 → 依赖已安装 → 开发服务器已启动 → 导航到 http://localhost:5173 → 页面加载完成
- **When**：点击顶部项目下拉框按钮 → 等待下拉菜单展开
- **Then**：下拉菜单中应显示"打开文件夹"按钮（带 FolderOpen 图标）✅

**测试用例 4.2：Node v24 下验证目标2 — 点击"打开文件夹"后成功添加项目** ✅
- **测试数据**：目标目录 `C:\Projects\forest`
- **Given**：下拉菜单已展开，"打开文件夹"按钮可见
- **When**：点击"打开文件夹"按钮 → ⏸️ 暂停等待原生对话框出现 → 用户手动选择 `C:\Projects\forest` 并确认 → 等待页面响应

## 验证方案
所有测试用例的 Then 步骤均验证通过即为通过。

## 操作规则
### 默认规则
1. 直接用浏览器进行端到端测试，不写任何脚本
2. 以本文档的测试步骤为操作流程依据
3. 输入框操作：输入完成后用回车键或点击"发送"/"提交"按钮，**禁止**用 `\n` 字符
4. 下拉选择：用 select 操作，不要手动模拟键盘
5. 按钮点击：直接用 click 操作定位按钮元素
6. 等待策略：操作后等待页面响应（await_element / await_text），不要假设即时生效

### 项目特定规则
1. 项目下拉框按钮使用 CSS 选择器 `button:has(svg.lucide-folder)` 定位（参考 base.md）
2. 点击"打开文件夹"后需暂停，等待用户手动在原生对话框中选择文件夹
3. 原生对话框操作不属于浏览器测试范畴，由用户手动完成
