# E2E 测试计划：Agent 输入框斜杠命令智能提示

## 测试简述

验证 Agent 聊天输入框中的斜杠命令智能提示功能：输入 `/` 后弹出命令列表，键盘导航选择命令后光标位置正确。

## 测试目标列表

- [ ] 目标1：输入 `/` 后弹出命令列表弹窗
- [ ] 目标2：Tab 键选择命令后，光标出现在命令后的空格之后（非命令之前）
- [ ] 目标3：Enter 键选择命令后，光标位置同样正确

## 测试方案

从首页开始，选择项目后进入 NewSessionPage 输入框，测试斜杠命令的完整交互流程。然后在 AgentPanel 中重复测试。

## 测试用例列表

### 目标 1：输入 `/` 后弹出命令列表弹窗

**测试用例 1.1：NewSessionPage 输入框**
- **测试数据**：项目 harnesson
- **Given**：导航到 http://localhost:5173 → 选择项目 harnesson → 输入框可见
- **When**：点击输入框 → 输入 `/`
- **Then**：应看到斜杠命令弹窗出现在输入框上方 → 弹窗中包含至少一个命令（如 /clear）

**测试用例 1.2：AgentPanel 输入框**
- **测试数据**：发送一条消息创建 Agent
- **Given**：在 NewSessionPage 输入框中输入 "hello" → Enter 发送 → 等待进入 AgentPanel 对话页
- **When**：点击 AgentPanel 底部输入框 → 输入 `/`
- **Then**：应看到斜杠命令弹窗出现在输入框上方

### 目标 2：Tab 键选择命令后光标位置正确

**测试用例 2.1：Tab 选择命令**
- **Given**：在输入框中输入 `/` → 弹窗出现
- **When**：按 Tab 键选择第一个命令
- **Then**：输入框文本变为 `/clear `（或选中的命令 + 空格）→ 光标位于命令文本的末尾（空格之后）→ 验证方式：输入任意文字如 "test"，应追加在命令空格之后

### 目标 3：Enter 键选择命令后光标位置正确

**测试用例 3.1：Enter 选择命令**
- **Given**：在输入框中输入 `/` → 弹窗出现
- **When**：按 Enter 键选择第一个命令
- **Then**：输入框文本变为 `/clear `（或选中的命令 + 空格）→ 光标位于命令文本的末尾

## 验证方案

所有测试用例的 Then 步骤均验证通过即为通过。光标位置验证方式：选择命令后输入额外文字，检查文字是否追加在命令之后。

## 操作规则

### 默认规则
1. 直接用浏览器进行端到端测试，不写任何脚本
2. 以本文档的测试步骤为操作流程依据
3. 输入框操作：输入完成后用回车键或点击"发送"按钮，禁止用 `\n` 字符
4. 下拉选择：用 select 操作
5. 按钮点击：直接用 click 操作
6. 等待策略：操作后 await_element / await_text

### 项目特定规则
1. dev server 端口：5173
2. 选择项目策略：eval 触发 React 点击打开下拉 → click 选择项目
