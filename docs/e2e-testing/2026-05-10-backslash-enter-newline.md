# E2E 测试计划：Agent 聊天输入框 `\` + Enter 换行功能

## 测试简述
验证 Agent 聊天输入框中，输入 `\` 后按 Enter 时，触发输入框内文本换行（并去掉 `\` 符号），而不是发送消息。同时验证普通 Enter 仍然发送消息、Shift+Enter 仍然换行。

## 测试目标列表
- [ ] 目标1：`\` + Enter 应删除 `\` 并插入换行，不发送消息
- [ ] 目标2：普通 Enter 仍然发送消息（回归测试）
- [ ] 目标3：Shift+Enter 仍然插入换行（回归测试）
- [ ] 目标4：`\` 后按 Enter 再继续输入并发送，消息内容正确

## 测试方案
从首页导航到 Agent 对话页面，在聊天输入框中测试各种 Enter 组合键行为。参考 base.md 中"进入首页"和"选择项目"策略。

## 测试用例列表

### 目标 1：`\` + Enter 应删除 `\` 并插入换行，不发送消息

**测试用例 1.1：输入文本后 `\` + Enter 换行**
- **测试数据**：输入文本 `hello` + `\` + Enter
- **Given**：导航到 http://localhost:5173 → 选择项目进入 Agent 对话页面 → 输入框可见且为空
- **When**：在输入框中输入 `hello` → 输入 `\` → 按 Enter
- **Then**：输入框中应显示 `hello` 后跟一个换行（`hello` 在第一行，光标在第二行）→ 输入框中不应存在 `\` 字符 → 消息不应被发送（聊天区域无新消息出现）

**测试用例 1.2：空输入后 `\` + Enter 换行**
- **测试数据**：仅输入 `\` + Enter
- **Given**：导航到 Agent 对话页面 → 输入框可见且为空
- **When**：在输入框中输入 `\` → 按 Enter
- **Then**：输入框中应出现空行（光标移到第二行）→ 不存在 `\` 字符 → 消息未被发送

### 目标 2：普通 Enter 仍然发送消息（回归测试）

**测试用例 2.1：输入文本后直接 Enter 发送**
- **测试数据**：输入文本 `test message`
- **Given**：导航到 Agent 对话页面 → 输入框可见且为空
- **When**：在输入框中输入 `test message` → 按 Enter
- **Then**：消息被发送（聊天区域出现新消息气泡）→ 输入框清空

### 目标 3：Shift+Enter 仍然插入换行（回归测试）

**测试用例 3.1：输入文本后 Shift+Enter 换行**
- **测试数据**：输入文本 `line1` + Shift+Enter + `line2`
- **Given**：导航到 Agent 对话页面 → 输入框可见且为空
- **When**：在输入框中输入 `line1` → 按 Shift+Enter → 输入 `line2`
- **Then**：输入框中应显示两行文本（`line1` 在第一行，`line2` 在第二行）→ 消息未被发送

### 目标 4：`\` 后按 Enter 再继续输入并发送，消息内容正确

**测试用例 4.1：换行后继续输入并发送**
- **测试数据**：输入 `hello` + `\` + Enter + `world` + Enter
- **Given**：导航到 Agent 对话页面 → 输入框可见且为空
- **When**：在输入框中输入 `hello` → 输入 `\` → 按 Enter（应换行）→ 输入 `world` → 按 Enter（应发送）
- **Then**：消息被发送 → 发送的消息内容应包含 `hello` 和 `world`（两行）→ 输入框清空

## 验证方案
所有测试用例的 Then 步骤均验证通过即为通过。

## 操作规则
### 默认规则
1. 直接用浏览器进行端到端测试，不写任何脚本
2. 以本文档的测试步骤为操作流程依据
3. 输入框操作：输入完成后用回车键或点击"发送"/"提交"按钮，**禁止**用 `\n` 字符
4. 下拉选择：用 select 操作，不要手动模拟键盘
5. 按钮点击：直接用 click 操作定位按钮元素
6. 等待策略：操作后等待页面响应（await_element / await_text），不要假设即时生效

### 项目特定规则
- 输入框为 contentEditable div，聚焦通过 click 输入框区域实现
- 换行验证通过 eval 检查输入框的 innerHTML 中的 `<br>` 标签实现
- 发送验证通过 eval 检查聊天区域的消息数量实现
