# E2E 测试计划：Specs Graph 框选蓝色矩形残留修复

## 测试简述
验证框选多个节点后松开鼠标，蓝色选择矩形是否消失。多次框选不同层级节点，确认选择矩形不残留。**核心验证方式：视觉截图，检查画布上是否存在任何蓝色/虚线矩形框（不仅限于 ReactFlow 官方选择框，任何样式相近的蓝色矩形都算残留）。**

## 测试目标列表
- [ ] 目标1：框选多节点后蓝色选择矩形消失，仅保留节点高亮
- [ ] 目标2：框选单个节点后选择矩形消失
- [ ] 目标3：连续多次框选，选择矩形均正常消失
- [ ] 目标4：框选拖拽过程中不改变当前选择，仅松开鼠标后选中被框住的节点

## 测试方案
首页 → 选择 harnesson 项目 → 导航到 Graph 页面 → 切换到 Specs Graph 标签，执行框选操作并通过截图视觉对比验证选择矩形是否残留。

## 测试用例列表

### 目标 1：框选多节点后选择矩形消失

**测试用例 1.1：框选多个同层节点**
- **测试数据**：harnesson 项目，Specs Graph 已加载
- **Given**：导航到 http://localhost:5173 → 选择 harnesson 项目 → 点击 Graph 标签 → 切换到 Specs Graph → 等待节点渲染
- **When**：在画布空白区域按住左键拖拽框选多个节点 → 松开鼠标
- **Then**：
  1. 被框选的节点显示选中高亮
  2. **截图画面，视觉确认页面上无任何残留蓝色/虚线矩形框**（检查整个画布区域，包括节点周围、边缘、空白处）
  3. 用 eval 检查 DOM 中无 `.react-flow__selectionpane` 可见选择框元素

**测试用例 1.2：框选跨层级节点**
- **测试数据**：同上
- **Given**：Specs Graph 已加载
- **When**：框选覆盖不同列（不同层级）的节点 → 松开鼠标
- **Then**：
  1. 被覆盖的节点高亮
  2. **截图画面，视觉确认页面上无任何残留蓝色矩形框**

### 目标 2：框选单节点后选择矩形消失

**测试用例 2.1：框选仅覆盖一个节点**
- **Given**：Specs Graph 已加载
- **When**：框选仅覆盖一个节点 → 松开鼠标
- **Then**：
  1. 该节点高亮
  2. DetailPanel 打开
  3. **截图画面，视觉确认页面上无任何残留蓝色矩形框**

### 目标 3：连续多次框选

**测试用例 3.1：连续三次不同区域框选**
- **Given**：Specs Graph 已加载
- **When**：框选区域A → 松开 → 截图验证 → 框选区域B → 松开 → 截图验证 → 框选区域C → 松开 → 截图验证
- **Then**：**每次松开后的截图中，页面上均无残留蓝色矩形框**，仅保留当前框选节点的高亮

### 目标 4：框选拖拽过程中不改变当前选择，仅松开鼠标后选中被框住的节点

本目标验证框选操作的原子性：拖拽期间 React Flow 节点的选中状态不应发生变化，所有选中变化应在 mouseup 之后一次性生效。

**测试用例 4.1：先选中节点再框选 — 拖拽过程中保留原选择**
- **测试数据**：harnesson 项目，Specs Graph 已加载
- **Given**：Specs Graph 已加载 → 单击选中一个节点（节点A 高亮）
- **When**：在画布空白区域按住左键开始框选拖拽（覆盖其他节点）→ **拖拽过程中（未松开鼠标）截图并 eval 检查当前选中状态**
- **Then**：
  1. 拖拽过程中，节点A 仍保持选中状态（高亮不变）
  2. 拖拽过程中，被框选覆盖的节点**尚未**被选中
  3. 松开鼠标后，仅被框选覆盖的节点被选中（节点A 不再选中，除非它在框选范围内）
  4. **松开后截图，视觉确认无蓝色矩形残留**

**测试用例 4.2：从空选择状态开始框选 — 拖拽过程中保持空选择**
- **测试数据**：同上
- **Given**：Specs Graph 已加载 → 单击画布空白区域清空所有选择
- **When**：在画布空白区域按住左键开始框选拖拽 → **拖拽过程中（未松开鼠标）截图并 eval 检查选中状态**
- **Then**：
  1. 拖拽过程中，所有节点保持未选中状态（无高亮）
  2. 松开鼠标后，被框选覆盖的节点被选中
  3. **松开后截图，视觉确认无蓝色矩形残留**

## 验证方案
- 所有 Then 步骤均通过即为修复成功
- **关键判定**：每次框选松开后截图，人眼观察画布内是否残留任何蓝色/虚线矩形 — 无论是否为 ReactFlow 官方选择框，只要视觉上存在蓝色矩形即为 bug 仍存在
- 辅助判定：eval 检查 `.react-flow__selectionpane` 是否存在可见子元素

## 操作规则
### 默认规则
1. 直接用浏览器进行端到端测试
2. 输入框用回车提交
3. 操作后等待页面响应
4. **每次框选松开后必须截图并读取查看，用视觉确认**

### 项目特定规则
1. 开发服务器：http://localhost:5173
2. 项目名：harnesson
3. ReactFlow 节点需完全渲染后再操作
4. 框选通过 drag 操作完成（从画布空白区域 mousedown → 拖拽覆盖节点 → mouseup）
5. **蓝色选择框/矩形**：泛指任何蓝色 (#0000ff / blue 系) 或虚线样式的矩形框，不限于特定 CSS class
