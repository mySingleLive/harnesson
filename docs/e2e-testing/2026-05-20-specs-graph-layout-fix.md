# E2E 测试计划：Specs Graph 布局修复

## 测试简述
修复 Specs Graph 页面布局的 5 个问题：节点重叠、节点不全、卡片过宽、方向应为水平(LR)、节点不可拖动。

## 测试目标列表
- [x] 目标1：布局方向从垂直(TB)改为水平(LR)，从左到右展开 ✅
- [x] 目标2：节点之间不重叠，间距合理 ✅
- [ ] 目标3：显示项目规格树所有节点（包括深层子节点）⚠️ 数据问题 — feature 级别节点尚未生成，需运行 sync-specs
- [x] 目标4：节点卡片宽度适中，不占用过多空间 ✅
- [x] 目标5：可通过鼠标拖动节点位置 ✅
- [x] 目标6：不同域的子节点用不同颜色，相同域（含子节点、孙节点）用同一种颜色 ✅
- [x] 目标7：任何不同节点卡片之间不重叠 ✅

## 测试方案
从首页导航到 Graph 页面 → 切换到 Specs Graph 标签 → 验证布局修复效果。

## 测试用例列表

### 目标 1：布局方向为水平 LR

**测试用例 1.1：验证树从左到右展开**
- **Given**：导航到 http://localhost:5173 → 选择 harnesson 项目 → 点击 Graph 导航项 → 切换到 Specs Graph 标签
- **When**：查看 Specs Graph 页面渲染完成
- **Then**：根节点（Harnesson）在最左侧 → 子节点在右侧展开 → 边连线从左到右连接

### 目标 2：节点不重叠

**测试用例 2.1：验证节点间距**
- **Given**：Specs Graph 页面已加载
- **When**：查看所有节点的布局
- **Then**：任何两个节点之间都有明显间距 → 节点文字完全可见 → 节点边界不重叠

### 目标 3：显示所有节点

**测试用例 3.1：验证所有层级节点可见**
- **Given**：Specs Graph 页面已加载
- **When**：查看图的完整内容（可能需要缩小）
- **Then**：显示根节点（Harnesson）→ 显示所有域节点（应用基础设施、规格图谱、项目管理、AI Agent 对话系统）→ 域节点下方显示对应的 Feature 子节点

### 目标 4：卡片宽度适中

**测试用例 4.1：验证节点卡片宽度**
- **Given**：Specs Graph 页面已加载
- **When**：查看节点卡片
- **Then**：卡片宽度适中（不超过 180px）→ 文字不被截断 → 卡片紧凑不臃肿

### 目标 5：节点可拖动

**测试用例 5.1：验证鼠标拖动节点**
- **Given**：Specs Graph 页面已加载
- **When**：鼠标按住一个节点 → 拖动到新位置 → 松开鼠标
- **Then**：节点跟随鼠标移动 → 松开后节点停留在新位置 → 边连线跟随移动

### 目标 6：域色区分

**测试用例 6.1：验证不同域使用不同颜色，同域节点颜色一致**
- **Given**：导航到 http://localhost:5173 → 选择 harnesson 项目 → 点击 Graph 导航项 → 切换到 Specs Graph 标签 → 等待页面渲染完成
- **When**：查看图中各域节点及其所有子节点、孙节点的颜色
- **Then**：根节点（项目）为紫色 → 每个域节点有自己独特的颜色（至少3种不同域色）→ 属于同一域的 Feature 子节点与其父域节点颜色一致 → 属于同一域的更深层级孙节点也与其根域节点颜色一致 → 不同域的子节点之间颜色明显不同

### 目标 7：节点卡片无重叠

**测试用例 7.1：验证所有节点卡片无重叠**
- **Given**：Specs Graph 页面已加载
- **When**：查看图中所有节点的位置和边界
- **Then**：任意两个节点卡片的边界框不重叠 → 节点间距清晰可辨 → 节点文字完全可见，不被其他节点遮挡

## 验证方案
所有 7 个测试用例的 Then 步骤均通过即为测试通过。

## 操作规则
### 默认规则
1. 直接用浏览器进行端到端测试，不写任何脚本
2. 以本文档的测试步骤为操作流程依据
3. 输入框操作：输入完成后用回车键或点击"发送"/"提交"按钮，禁止用 `\n` 字符
4. 等待策略：操作后等待页面响应（await_element / await_text），不要假设即时生效

### 项目特定规则
1. 开发服务器地址：http://localhost:5173
2. 项目名称：harnesson
3. ReactFlow 图组件需等待节点完全渲染后再验证
