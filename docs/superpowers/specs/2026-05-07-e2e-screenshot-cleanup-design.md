# 设计：E2E 测试截图文件自动清理

## 背景

E2E 测试过程中通过浏览器 `screenshot` 操作生成大量截图文件（`.png` 扩展名和无扩展名的 PNG 文件），保存在项目根目录。当前 skill 没有清理机制，导致测试后项目根目录残留数十个截图文件。

## 修改目标

修改 `.claude/skills/e2e-testing/SKILL.md`，在测试结束后自动删除本次测试中生成的截图文件。

## 方案：标记文件 + 增量清理

### 原理

在测试开始时创建标记文件 `.e2e-start`，测试结束后删除项目根目录下所有比标记文件更新的 PNG 截图文件。

### 修改点

#### 1. 阶段二 Step 2：开发服务器管理

在服务器检测之前，新增创建标记文件的指令：

```bash
touch .e2e-start
```

此文件作为后续清理的时间基准。

#### 2. 阶段二 Step 5（新增）：清理截图文件

在终止条件（所有测试通过或测试终止）之后，新增清理步骤：

```bash
find . -maxdepth 1 -type f \( -name '*.png' -o ! -name '*.*' \) -newer .e2e-start -delete && rm .e2e-start
```

清理范围：
- 匹配：`.png` 扩展名的文件 + 无扩展名的文件
- 仅删除比 `.e2e-start` 更新的文件（即本次测试中生成的）
- 最后删除 `.e2e-start` 标记文件

不清理的内容：`.md`、`.html`、`.txt` 等其他浏览器生成的文件。

### 触发时机

在以下两种终止情况下都执行清理：
1. 所有测试用例执行完毕且全部通过 → 清理 → 向用户报告测试结果摘要
2. 层级 3 代码修复同一问题 3 次仍失败 → 清理 → 升级给用户

### 安全性

- 仅删除项目根目录（`-maxdepth 1`）下的文件，不递归
- 通过 `-newer .e2e-start` 确保只删除本次测试生成的文件
- 不影响 `docs/`、`apps/` 等子目录中的文件
