# Specs Tree Data Source Redesign

**Date:** 2026-05-20
**Status:** approved
**Scope:** specs-graph, specs-list, specs-document tabs only

## Summary

Replace the current flat `graph.json`/`list.json` data source with the hierarchical `specs/nodes/` directory tree generated by `sync-specs`. Only specs tabs are affected; architect tabs continue reading `.harnesson/architecture/`.

## Architecture

```
.harnesson/specs/
  project.json          ← root node
  nodes/
    ai-agent/node.json  ← child nodes
    project-management/node.json
    ...

Server                          Frontend
------                          --------
specs/tree API  ──GET──→  graphStore.loadSpecsTree()
  reads nodes/*      →  specsTree (root)
  returns root +     →  specsNodeMap (id → node)
  node map                ↓
                    SpecsGraph  ← buildGraphFromTree()
                    SpecsList   ← sorted nodeMap values
                    SpecsDocument ← assembled markdown
                    DetailPanel ← nodeMap lookup
```

## New Types

```typescript
// packages/shared/src/types/spec-node.ts (replace current)

interface AcceptanceCriterion {
  given: string;
  when: string;
  then: string;
}

interface SpecTreeNode {
  id: string;
  name: string;
  level: number;
  parent: string | null;
  children: string[];
  isLeaf: boolean;
  summary: string;
  goals: string[];
  acceptanceCriteria: AcceptanceCriterion[];
  status: string;  // draft | review | done
  design: unknown;
}
```

## New API

`GET /api/specs/tree?projectPath=...`

Response:
```json
{
  "root": { /* SpecTreeNode */ },
  "nodes": { "id": { /* SpecTreeNode */ }, ... }
}
```

Reads `project.json` as root, then recursively reads `nodes/{childId}/node.json`. Skips missing child nodes with console warning. Returns root + flat map for frontend lookup.

## Frontend Changes

### graphStore additions

```typescript
specsTree: SpecTreeNode | null;
specsNodeMap: Record<string, SpecTreeNode> | null;
loadSpecsTree(projectPath: string): Promise<void>;
```

Existing `specsData`/`architectData` unchanged — two data paths coexist.

### Node card design

Each node card shows three elements: status badge (colored dot + text), title, and one-line truncated summary.

Status color map:
- `draft` → yellow (#f9e2af)
- `review` → blue (#89b4fa)  
- `done` → green (#a6e3a1)

Type mapping from level: 1 → project, 2-3 → domain, 4+ → feature.

### Component data sources

| Component | Data | Notes |
|-----------|------|-------|
| SpecsGraph | specsTree | buildGraphFromTree() → nodes + edges |
| SpecsList | specsNodeMap | flat values sorted by level |
| SpecsDocument | specsNodeMap | markdown assembled per level |
| DetailPanel | specsNodeMap | full fields: goals, AC, children |
| ArchitectGraph | architectData (unchanged) | |
| TechnicalDocument | architectData (unchanged) | |

### GraphPage

`loadGraph` + `loadSpecsTree` called in parallel. Specs tabs use new data; architect tabs use old.

## Edge Cases

| Scenario | Behavior |
|----------|----------|
| project.json missing | Show SyncView with sync prompt |
| Child node.json missing | Skip node, console.warn, continue rendering |
| Optional fields empty | Omit corresponding UI sections, no error |
| Unknown status value | Render without status badge |
| Empty children + isLeaf | Normal leaf node, no edges |

## Testing

- **Backend:** specs/tree returns correct data; empty dir; missing child; invalid JSON
- **graphStore:** loadSpecsTree parses tree, buildGraphFromTree edge conversion
- **GraphNodes:** status badge color rendering
- **DetailPanel:** goals, AC display
- **SpecsGraph:** renders with new data; null → empty state
- **SpecsList:** level sorting; status column
