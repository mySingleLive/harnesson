# Tool Card Preview & Agent Sub-Agent Tree Rendering

Date: 2026-05-04

## Problem

Tool call cards (Glob, Read, Bash, etc.) show blank content in collapsed state — users must expand to see any output. Agent sub-agent calls are rendered by GenericCard with no useful info, no way to see what the sub-agent did internally.

## Goals

1. Tool cards show a 1–2 line content preview in collapsed state
2. Agent sub-agent calls show description, model, and expandable tree of internal events

## Design

### Part 1: Tool Card Collapsed Preview

Add a preview line below the existing summary in each tool card. Visible in both collapsed and expanded states. Uses `text-gray-600` to distinguish from summary text.

| Tool | Existing summary | New preview |
|------|-----------------|-------------|
| Glob | `🔍 Glob · **/*.ts` | First 2 matched file paths |
| Read | `📄 Read · src/foo.ts` | First 2 lines of file content (with line numbers) |
| Bash | `⚡ Bash · npm run build` | First 1–2 lines of output |
| Grep | `🔎 Grep · "pattern"` | First 2 match lines |
| Write | `✏️ Write · src/foo.ts` | First 2 lines of content |
| Edit | `📝 Edit · src/foo.ts` | `old_string` first line → `new_string` first line |

Implementation: each card component adds a preview `<div>` between the summary button and the collapsible detail area. No changes to `CollapsibleCard`.

### Part 2: Agent Sub-Agent Card

#### Data Model

`PairedToolEvent` adds:

```typescript
interface PairedToolEvent {
  tool: string;
  input: Record<string, unknown>;
  output?: string;
  isError?: boolean;
  duration?: number;
  subEvents?: PairedToolEvent[];  // nested tool events from sub-agent
  subTexts?: string[];            // text segments generated by sub-agent
}
```

`AgentStreamEvent` adds:

```typescript
interface AgentStreamEvent {
  // existing fields...
  parentToolUseId?: string;  // which Agent tool_use this event belongs to
  depth?: number;            // nesting level (0=top, 1=sub-agent, 2=sub-sub)
}
```

#### Backend: Stack-Based Nesting Tracker

In `claude-code-adapter.ts`, track Agent nesting with a stack:

```
state:
  agentStack: string[]        // stack of active Agent tool_use IDs
  subEventBuffers: Map<string, { texts: string[], toolEvents: PairedToolEvent[] }>

on Agent tool_use:
  push tool_use_id onto agentStack
  create buffer entry for this ID

on Agent tool_result:
  pop agentStack
  flush buffer into PairedToolEvent.subEvents / subTexts
  attach to the parent Agent event

on other tool_use / tool_result / text:
  if agentStack non-empty:
    attach event to buffer of agentStack top
  else:
    emit as top-level event (current behavior)
```

This supports arbitrary nesting depth (sub-agents calling sub-agents).

#### Frontend: AgentCard Component

**Collapsed state:**

```
▸ 🤖 Agent · "探索项目结构"  sonnet ✓ 12s
```

Fields:
- Icon: `🤖`
- Description: from `input.description`
- Model badge: from `input.model` (sonnet / opus / haiku)
- Status: success (✓) or error (✗) + duration

**Expanded state — recursive tree:**

```
▾ 🤖 Agent · "探索项目结构"  sonnet ✓ 12s
  │ 💬 "我来帮你探索这个项目的结构..."
  │ ▸ 🔍 Glob · **/*.ts               ✓ 15 files
  │ ▸ 📄 Read · src/index.ts           ✓
  │ 💬 "这个项目使用了..."
```

Rendering:
- Left border: vertical line `│` in `border-l-2 border-gray-700`
- Text segments: `<div>` with `💬` icon, `text-gray-400`, italic
- Tool events: reuse `SingleToolEventCard`, indented one level
- Recursive: if a sub-event is also an Agent, nest another level with deeper indent

Component structure:

```
AgentCard.tsx
  CollapsibleCard (outer)
    summary: description + model badge
    children: AgentEventTree
      AgentEventTree.tsx (recursive)
        for each subText: render text segment
        for each subEvent: render SingleToolEventCard or AgentCard (recursive)
```

#### Tool Card Map Update

Add `Agent` to `toolCardMap` in `ToolEventCard.tsx`:

```typescript
const toolCardMap = {
  // existing...
  Agent: AgentCard,
};
```
