# Sync-Specs 性能优化设计

## 问题诊断

sync-specs 技能（SKILL.md 500 行 + references 397 行 = 897 行）存在以下瓶颈：

1. **3 个嵌套"循环直到通过"结构**（Step 2.5/4/5.0），最坏情况相互触发 O(n²)
2. **5 项重复检查**：4.3a/3b 与 5.0.4 重叠；4.3c 与 Step 3d 冗余；4.4 与 2.5.3 重叠
3. **Step 3 有 5 个子步骤**（3a-3e），各自独立读文件
4. **897 行指令**全部加载到上下文，模型处理开销大

## 优化方案

### 流程重构：8 步 3 循环 → 5 步 1 循环

```
优化前：Step1 → Step2 → Step2.5♻️ → Step3(5子步骤) → Step4♻️(6项) → Step5.0♻️(4项,回退Step4) → Step5.1 → Step6

优化后：Step1(扫描+分析) → Step2(生成草稿,2子步骤) → Step3(统一校验♻️,6项) → Step4(用户审核) → Step5(转正)
```

### Step 1：扫描 + 变更分析（合并原 Step 1 + 2 + 2.5）

- 读取项目配置、检测模式（增量/全量）
- 增量：git diff 识别变更 → 匹配节点 → 5 项完整性审计
- 全量：扫描全部源文件 → 递归识别业务节点
- 自检在思维中完成（代码一致性、叶子节点判定、结构合理性）
- 变更清单以树形文本存入 `draft/README.md`，不单独写 sync-plan 文件

### Step 2：生成草稿（5 子步骤 → 2 子步骤）

**2a — 生成节点 JSON**：分析 sourceFiles → 生成 summary/goals/acceptanceCriteria/testCases/specDetail/constraints → 写入 draft/nodes/

**2b — 生成设计文档**：检测节点类型 → 按模板生成内容 → 追加 specDetail/constraints 章节 → 写入 draft/design/ → 设置 design 字段

补全逻辑（增量）内嵌为 2a 的条件分支。

### Step 3：统一校验（合并原 Step 4 + 5.0，10 项 → 6 项）

发现失败项 → 修复当前节点 → 继续下一个 → 全部完成后仅复查修复过的项。无回退到 Step 2。

| # | 检验项 | 来源 |
|---|--------|------|
| 1 | 格式校验 | 原 4.1 |
| 2 | 版本校验 | 原 4.2 |
| 3 | 内容校验（含 specDetail/constraints 完整性） | 合并 4.3 + 4.3a + 4.3b + 5.0.4 |
| 4 | 唯一性校验 | 原 4.5 |
| 5 | 设计文档校验（含 JSON↔MD 一致性） | 合并 4.6 + 4.3c |
| 6 | 覆盖度审查（验收标准 + 测试用例 + 子节点完整性） | 合并 5.0.1 + 5.0.2 + 5.0.3 |

删除的重复检查：
- 原 4.4（结构合理性）→ Step 1 自检已保证
- 原 4.3a/3b 独立检查 → 合并到第 3 项
- 原 5.0.3 独立检查 → 合并到第 6 项

### Step 4：用户审核（精简）

展示变更摘要、校验结果、新旧 diff，等待确认。

### Step 5：转正（精简）

移动 draft/ → specs/，清理 draft/，更新 lastSyncAt。

### SKILL.md 篇幅精简

| 区域 | 当前 | 目标 | 方式 |
|------|------|------|------|
| 节点识别原则 | ~90 行 | ~35 行 | 详细示例移到 references |
| Step 1 | ~30 行 | ~25 行 | 合并原 Step 2+2.5 |
| 原 Step 2+2.5 | ~55 行 | 删除 | 合并到 Step 1 |
| Step 2 (原 Step 3) | ~75 行 | ~40 行 | 5→2 子步骤 |
| Step 3 (原 Step 4+5.0) | ~90 行 | ~50 行 | 10→6 项，去重 |
| Agent 行为规则表格 | ~12 行 | 删除 | 各步骤已包含 |
| 其他 | ~148 行 | ~90 行 | 去冗余表述 |

### 增量模式优化

- 未变更节点不读取 JSON 文件内容（仅读 children 数组）
- 未变更节点的设计文档不读取

## 预期效果

- 指令篇幅：-52%
- 校验循环：3 → 1
- 重复检查：5 → 0
- 草稿子步骤：5 → 2
- 增量模式文件读取量：全量 → 仅变更节点
- 整体提速：40-55%
